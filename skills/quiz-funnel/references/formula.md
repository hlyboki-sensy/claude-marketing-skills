# Формула патернів квіз-воронки v2.0

Зворотно спроєктовано з NeuroSamurai, Noom, Focus & Flow, Done., BedGear та 15+ інших висококонверсійних квіз-воронок. Доповнено дослідженнями ADHD-UX, гейміфікації та інтерактивних лендингів.

## 10 патернів висококонверсійного квіза

### 1. "Чат-друг, а не анкета"
Квіз імітує діалог у месенджері. Чат-бульбашки перед питаннями створюють відчуття спілкування з тим, хто розуміє. Знижує відчуття "мене тестують", підвищує чесність відповідей.

Реалізація: компонент ChatBubble перед першими 3 екранами. Сірий фон, скруглені кути, текст від першої особи.

### 2. "Техно-фреймінг"
Квіз подано як діагностику/сканування/аналіз. Підзаголовки змінюються: "Ініціалізація...", "Оцінка рівня...", "Аналіз...", "Калібрування...". Перетворює опитування на "медичну процедуру" і підвищує сприйману цінність.

Реалізація: заголовок у caps lock + змінний підзаголовок + прогрес-бар з відсотком.

### 3. Кольорова шкала тяжкості
Кожен варіант відповіді маркований кольоровою крапкою (🟢🟡🟠🔴💀). Візуально показує "ступінь ураження". Автоматично підсилює усвідомлення проблеми без довгих пояснень.

Реалізація: кольорове коло 12px зліва від тексту варіанта.

### 4. Чергування типів питань
- Single-select із 4 варіантами (6 питань) — швидко, не вантажить
- Multi-select із 6-10 варіантами (2 питання) — залучає глибше, людина сама "ставить діагноз"
- Вставки-перепочинки між блоками — валідація + факт + надія

Реалізація: 3 типи компонентів питань + InsightScreen.

### 5. Емоційна дуга
Порядок екранів створює керовану емоційну траєкторію:
1. Цікавість (вхід)
2. Впізнавання себе (перші питання)
3. Наростаючий дискомфорт (симптоми, втрати)
4. Надія (вставка на середині з науковим фактом)
5. Глибоке усвідомлення (калькулятор шкоди)
6. Шок від цифр (490 ДНІВ)
7. Рішення / вихід (продукт як порятунок)

### 6. Персоналізований калькулятор шкоди
Відповідь на кількісне питання використовується для розрахунку:
- Втрата часу: години/день → тижні → місяці → роки → дні життя
- Втрата грошей: години × ставка ЦА → за рік → за 5 років
- Альтернативи: "що можна було б зробити за цей час"

Це ОСОБИСТО ЙОГО цифри, не абстрактна статистика.

### 7. Кваліфікаційне фінальне питання
Останнє питання сегментує лідів за "температурою":
- 🔥 Гарячий → тиснути на терміновість
- ⚡ Теплий → показати шлях
- 🌊 Потребує надії → акцент на підтримку
- 🤔 Холодний → ретаргетинг

### 8. "Дзеркало проблеми" (НОВЕ)
Питання служать не для збору даних, а для того, щоб користувач УСВІДОМИВ свою проблему. Коли він відповідає "так, я втрачаю фокус кожні 10 хвилин" — він фіксує проблему у свідомості. До квіза проблема абстрактна. Після — конкретна і вимірна.

Noom використовує цей принцип у 67-кроковому квізі з $500M+ виручки.

Реалізація: кожне питання формулюй як "дзеркало" — користувач бачить себе у варіантах відповіді.

### 9. Cadence 3:1 (НОВЕ, Noom-стиль)
Кожні 3 питання — 1 мотиваційна вставка:
- Соціальний доказ ("89% людей з такими симптомами...")
- Графік прогресу ("Ти вже на 60% — лишилося трохи")
- Зняття провини ("Це не лінь — це нейрохімія")
- Міні-інсайт ("Чи знав ти, що...")

Створює відчуття діалогу, а не допиту. Суперечить пораді "роби коротше" — довга воронка конвертує КРАЩЕ, якщо кожен крок простий.

Реалізація: InsightScreen через кожні 3 питання. Різні типи вставок чергуються.

### 10. "Yes Ladder" (НОВЕ, Noom-стиль)
Порядок питань за наростанням залученості:
1. Легкі / нейтральні (вік, формат, досвід)
2. Особисті / про проблему
3. Емоційні / про біль і втрати
4. Про готовність / про гроші та дії

Кожне "так" робить наступне "так" легшим. До моменту paywall користувач уже інвестував 5-15 хвилин → sunk cost effect.

## Повна карта екранів: Формат A (Діагностика)

```
ФАЗА 1: ВХІД (2-3 екрани)
├── Екран 1: Кваліфікаційне питання-фільтр (чат-бульбашка + 3 варіанти)
├── Екран 2: Валідація + емпатія (реакція на відповідь)
└── Екран 3: Підготовка до "сканування"

ФАЗА 2: ДІАГНОСТИКА (8 питань + 1-2 вставки)
├── Питання 1: Щоденна поведінка (single, 4 варіанти)
├── Питання 2: Кількісний маркер (single, 4-5 варіантів) ← для калькулятора
├── Питання 3: Симптоми (multi, 7-10 варіантів)
├── ВСТАВКА: Валідація + науковий факт + надія
├── Питання 4: Мотивація до дії (single, 4 варіанти)
├── Питання 5: Втрати за рік (multi, 5-7 варіантів)
├── [ВСТАВКА (опц.): Social proof / міні-статистика]
├── Питання 6: Провокаційний сценарій (single, 4 варіанти)
├── Питання 7: Поточний стан (single, 4 варіанти)
└── Питання 8: Готовність / кваліфікація (single, 4 варіанти)

ФАЗА 3: РЕЗУЛЬТАТИ (4 екрани)
├── Дашборд: скор + 3 метрики + рекомендація
├── Калькулятор: час + гроші + приклади
├── Якість життя: 5 наслідків
└── Шокове число: X ДНІВ за 10 років

ФАЗА 4: ПРОДУКТ (2-3 екрани)
├── Трансформація: "з X — до Y" + 4 обіцянки
├── Докази: метод + статистика
└── [Social proof (опц.): відгуки / лічильник]

ФАЗА 5: КОНВЕРСІЯ (2-3 екрани)
├── Фінальний CTA: логотип + слоган + кнопка + дефіцит
├── Форма: 2 поля + кнопка
└── [Achievement (опц.): "Ти розблокував статус [X]!" + share]
```

## Повна карта екранів: Формат B (Типологія)

```
ВХІД (2 екрани)
├── Hook: "Який у тебе [X]-стиль?" + "Дізнайся за 3 хвилини"
└── Фреймінг: "Немає правильних відповідей"

ПИТАННЯ (5-6 штук)
├── Питання 1: Стиль організації
├── Питання 2: Фокус і мотивація
├── Питання 3: Реакція на стрес
├── [ВСТАВКА: "Бачимо патерн..." + анімація обробки]
├── Питання 4: Гіперфокус / відновлення
├── Питання 5: Ухвалення рішень
└── [Питання 6 (опц.): Соціальна поведінка]

РЕЗУЛЬТАТ (3-4 екрани)
├── Reveal: "Ти — [Архетип]!" + анімація
├── Профіль: сильні сторони + сліпі зони
├── Стратегія: 3 персональні кроки
└── Share + CTA: картка для соцмереж + e-mail capture / продукт

КОНВЕРСІЯ (1-2 екрани)
├── Офер, адаптований під архетип
└── Форма
```

## Повна карта екранів: Формат C (Ультра-скринінг)

```
ВХІД (1 екран)
└── "[Проблема]? Дізнайся за 1 хвилину" + social proof

ПИТАННЯ (3-4 штуки)
├── Питання 1: головний симптом
├── Питання 2: частота / інтенсивність
├── Питання 3: вплив на життя
└── [Питання 4 (опц.): готовність до дії]

РЕЗУЛЬТАТ (1-2 екрани)
├── Світлофор: 🟢🟡🔴 + рекомендація
└── CTA + форма
```

## Повна карта екранів: Формат D (Калькулятор-шокер)

```
ВХІД (1 екран)
└── "Скільки коштує твоя [проблема]?"

СЛАЙДЕРИ (1-3 екрани)
├── Слайдер 1: основний параметр
├── Слайдер 2: вік/час
└── Слайдер 3: грошовий параметр

РЕЗУЛЬТАТ — ШОК (2-3 екрани)
├── 3 шок-числа (анімовані count-up)
├── Before/After візуалізація
└── Альтернативи: "За цей час можна було..."

[SCROLLYTELLING (опц., 5-6 кроків)]
├── "Зараз" → "День 3" → "День 7" → "День 14" → "День 30" → CTA

КОНВЕРСІЯ (2 екрани)
├── CTA + дефіцит
└── Форма
```

## Дизайн-референс (дефолтний)

- Фон: #000000 або #0A0A0A
- Акцент: #00FF88 (яскраво-зелений)
- Негативні метрики: #FF4444 (червоний), #FF8800 (помаранчевий)
- Попередження: #FFD700 (жовтий)
- Текст: #FFFFFF (основний), #999999 (другорядний)
- Картки: #1A1A2E або #111827 з border 1px solid #333
- Обрана картка: border-color: #00FF88, glow: box-shadow 0 0 20px rgba(0,255,136,0.3)
- Зелена картка (інсайт): фон #00FF8820, border #00FF88
- Кнопка CTA: фон #00FF88, текст #000000, border-radius 12px, glow: 0 0 30px rgba(0,255,136,0.4)
- Контейнер: max-width 640px, margin auto, padding 20px
- Шрифт: Inter, system-ui, -apple-system
- Прогрес-бар: height 4px, position fixed top, gradient green, transition 0.3s
- Анімації переходів: Framer Motion — fade + slideUp, duration 0.3-0.5s
- Stagger children: delay 0.05-0.1s між елементами

Альтернативні дизайн-системи — див. `knowledge-base.md` секція 4.
