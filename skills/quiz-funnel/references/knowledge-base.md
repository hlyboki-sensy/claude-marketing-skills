# База знань: Конверсійні патерни, ADHD-UX і дизайн-системи

## 1. Бібліотека CTA-формул

### CTA від першої особи (збільшують CTR)
Використання першої особи створює agency — "я ухвалюю рішення", а не "мені продають".

**Шаблони:**
- "Почати мою перезавантаження" (first person + action)
- "Дізнатися мій план відновлення" (curiosity + персоналізація)
- "Забронювати місце безкоштовно" (zero risk + exclusivity)
- "Розблокувати мій результат" (gamification + curiosity gap)
- "Отримати мою стратегію" (ownership + value)
- "Приєднатися до [N] учасників" (social proof + action)
- "Почати безкоштовний тест" (low friction)
- "Показати мій план" (curiosity)
- "Записатися першим" (exclusivity + urgency)
- "Забрати мій бонус" (value + ownership)

### Анти-патерни (НЕ використовувати):
- "Надіслати" / "Submit" — безлике
- "Зареєструватися" — надто формальне
- "Купити" — передчасне
- "Підписатися" — асоціація із зобов'язаннями

### Verbatim repetition
Повторення ключового повідомлення заголовка на кнопці CTA збільшує конверсію на 10+ п.п.
Приклад: Заголовок "Перезавантаж свій мозок" → Кнопка "Почати перезавантаження"

## 2. Конверсійні механіки

### Waitlist з позицією в черзі (модель Robinhood)
- Після підписки показати: "Ти #347 у черзі"
- "Запроси друга — піднімися вище"
- Створює вірусну петлю (Robinhood: 1M+ waitlist)
- Інструменти: LaunchList, Prefinery, Waitlister, QueueForm

### Milestone rewards (модель Duolingo)
- "Запросив 3 друзів → безкоштовний місяць"
- "10 друзів → lifetime доступ"
- Бейджі збільшують реферали на 116%

### Early Bird
- "Першим 500 — 50% знижка назавжди" з реальним лічильником
- Створює urgency + exclusivity
- Лічильник має бути РЕАЛЬНИМ (не фейковим — аудиторія чутлива)

### Countdown Timer
- "Передзапис закривається через 48:00:00"
- УВАГА: для ADHD-аудиторії таймери можуть викликати тривожність
- Використовуй м'яку версію: "Набір до [дата]" замість зворотного відліку

### Achievement після квіза
- "Ти розблокував статус [Назва]!"
- Шеринг у Stories — драйвер віральності
- "Збережи свій Score і відстежуй прогрес" (endowment effect)

### Social Proof (патерни)
- Live-лічильник: "Уже записалися: 1,247" (оновлюється)
- Цитати учасників (1-2 короткі)
- Логотипи (якщо B2B): "Довіряють: [лого]"
- Число: "60,000+ користувачів" (Done.)
- Рейтинг: "4.8/5 задоволеність"

### Lead capture після залучення (модель ADDA)
- E-mail запитується ПІСЛЯ проходження тесту, коли користувач хоче побачити результат
- Підвищує конверсію до 30%+
- "Надішлемо повний результат на e-mail" — обмін цінності на контакт

## 3. ADHD-Friendly UX принципи

### Критичні правила (застосовуй ЗАВЖДИ для wellness/mental health ніші)

| Принцип | Реалізація | Чому працює |
|---------|-----------|-------------|
| Одне питання на екран | Typeform-стиль | 60% вищий response rate |
| Progress bar завжди видно | Sticky top bar | +14-42% до конверсії |
| Chunking (max 7 елементів) | Закон Міллера | Netflix/Etsy обмежують до 6 |
| Progressive disclosure | Інформація в міру потреби | Знижує когнітивне навантаження |
| Миттєвий зворотний зв'язок | Анімація + haptic при виборі | Мікро-дофамін за кожну дію |
| Мінімум тексту | Одне речення на екран | Знижує візуальну втому |
| Спокійні візуали | Пастельні тони, мало анімацій | Overstimulation — ворог ADHD |
| Без таймерів | Самостійний темп | Таймери викликають тривожність |
| Auto-save | Можна повернутися | Час заповнення -50% |
| Sticky CTA | Кнопка завжди видно | +10% конверсія, 2x pages/session |
| < 3 сек до першої взаємодії | Перший елемент без скролу | 53% ідуть при >3 сек завантаження |
| Один CTA на сторінку | Одна ціль | Закон Гіка: множина = параліч |

### Що НЕ робити для ADHD-аудиторії
- Суворі таймери/дедлайни — посилюють тривожність
- Жорстке покарання (HP loss, дерева, що вмирають) — демотивує
- Перевантажений UI (безліч елементів, миготливі анімації)
- Висококонкурентні лідерборди — стрес через rejection sensitivity
- Забагато опцій одночасно — параліч вибору
- Пасивний контент без взаємодії

### Елементи, ідеальні для ADHD
- Миттєвий візуальний зворотний зв'язок (кожні 15 хвилин = мікро-анімація)
- Візуальна часова шкала (кільцевий таймер, радіальний прогрес)
- Персоналізовані нагороди (користувач сам визначає "призи")
- Новизна й unpredictability (рандомні бонуси, сезонні івенти)
- Гнучкий темп без fail state ("Freeze" стріка без втрати прогресу)

## 4. Дизайн-системи для квіз-лендингів

### Система A: Dark High-Contrast (стиль NeuroSamurai)
- Фон: #000000 або #0A0A0A
- Акцент: #00FF88 (яскраво-зелений)
- Негативні метрики: #FF4444 (червоний), #FF8800 (помаранчевий)
- Попередження: #FFD700 (жовтий)
- Текст: #FFFFFF (основний), #999999 (другорядний)
- Картки: #1A1A2E або #111827, border 1px solid #333
- Обрана картка: border-color: #00FF88, glow: box-shadow 0 0 20px rgba(0,255,136,0.3)
- Зелена картка (інсайт): фон #00FF8820, border #00FF88
- CTA кнопка: фон #00FF88, текст #000, radius 12px, glow: 0 0 30px rgba(0,255,136,0.4)
- Шрифт: Inter, system-ui, -apple-system
- Найкраще для: tech, biohacking, детокс, СДУГ, гейміфікація

### Система B: Calm Wellness (стиль Headspace/Calm)
- Фон: #F8F6F4 (теплий білий) або #FAFAFA
- Акцент: #6C63FF (м'який фіолетовий) або #5B86E5 (спокійний синій)
- Позитив: #4CAF50 (зелений)
- Негатив: #FF6B6B (м'який червоний)
- Текст: #2D3436 (майже чорний), #636E72 (сірий)
- Картки: #FFFFFF, radius 16px, shadow: 0 2px 12px rgba(0,0,0,0.08)
- Обрана картка: border 2px solid акцент, легкий shadow
- CTA кнопка: gradient або solid акцент, radius 24px (pill-shape)
- Шрифт: DM Sans, Nunito, або Inter
- Найкраще для: терапія, медитація, self-care, жіноча аудиторія

### Система C: Energy Business (фінанси/продуктивність)
- Фон: #0F0F1A (deep navy)
- Акцент: #FFD700 (золотий) або #FF6B35 (помаранчевий)
- Градієнти: linear-gradient(135deg, #667eea 0%, #764ba2 100%) для карток
- Текст: #FFFFFF, #A0A0B0
- Картки: rgba(255,255,255,0.05), backdrop-blur, border rgba(255,255,255,0.1)
- Glassmorphism: backdrop-filter: blur(10px)
- CTA кнопка: gradient золотий, font-weight 700
- Найкраще для: бізнес, фінанси, продуктивність, чоловіча аудиторія

### Система D: Nature/Organic (eco/wellness)
- Фон: #F5F0EB (бежевий)
- Акцент: #2D6A4F (темно-зелений)
- Вторинний: #95D5B2 (м'ятний)
- Текст: #1B4332, #52796F
- Картки: #FFFFFF, border-radius 12px
- Ілюстрації: line-art, ботанічні мотиви
- Шрифт: Lora (serif) для заголовків, Inter для тексту
- Найкраще для: екологія, здоров'я, харчування, натуральні продукти

### Загальні компоненти (для всіх систем)

**Progress Bar:**
- Height: 4px, position: fixed top
- Gradient або solid акцент
- Анімація: transition width 0.3s ease

**Chat Bubble:**
- Фон: rgba(255,255,255,0.1) для темних тем або #F0F0F0 для світлих
- Border-radius: 18px 18px 4px 18px (хвостик праворуч)
- Padding: 12px 16px
- Шрифт: на 2px менший за основний

**Option Card:**
- Min-height: 56px
- Padding: 16px
- Border-radius: 12px
- Transition: all 0.2s ease
- При виборі: scale(1.02), border-color: акцент, glow

**CTA Button:**
- Width: 100% (усередині контейнера)
- Height: 56px мінімум
- Border-radius: 12px (або 28px для pill)
- Font-weight: 600-700
- Letter-spacing: 0.5px
- Glow-ефект: box-shadow з акцентним кольором, opacity 0.3-0.4

**Анімації (Framer Motion):**
- Переходи між екранами: fade + slide up (duration 0.3-0.5s)
- Поява елементів: stagger children (0.05-0.1s між елементами)
- Вибір варіанта: scale(1.02) + border glow
- Шок-числа: count-up animation від 0 до значення (1.5-2s)
- Progress bar: spring animation

## 5. Психологія "Aha-Moment"

### Що створює Aha-момент
- "First win за 60 секунд" — завершити дію й отримати цінність до запиту даних
- "Дзеркало проблеми" — користувач УСВІДОМЛЮЄ проблему через питання ("так, я втрачаю фокус кожні 10 хвилин")
- "Персоналізований результат" — не "наш продукт хороший", а "ось що з ТОБОЮ і як змінити"
- "Рефреймінг" — до квіза проблема абстрактна, після — конкретна й вимірна

### Nudge після Aha-моменту
Одразу після aha-moment — чіткий CTA. Затримка навіть на 1 екран знижує конверсію.
Приклад: Шок-число → одразу кнопка "Почни перезавантаження"
