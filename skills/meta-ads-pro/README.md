# meta-ads-pro

**AI-таргетолог по Meta (Facebook / Instagram)** на актуальной (post-Andromeda) методологии **Ben Heath** — в виде **скилла для Claude Code**. Анализирует рекламный кабинет, креативы, видео, лендинги и настройки → выдаёт **действенный отчёт** и безопасно оптимизирует.

**Это самый полный и актуальный разбор методологии таргетинга в Meta, упакованный в рабочий AI-инструмент** — который не просто советует, а анализирует данные и действует. И не выдумывает цифр.

## 📚 База знаний — что внутри

**168 000 слов** структурированных знаний по актуальной методологии **Ben Heath**: **47 файлов знаний в 13 категориях** + 6 playbook'ов + 2 контура безопасности. Это не «набор советов», а многоуровневая база, где у каждого утверждения есть источник и дата.

### Формат каждого файла (единый, §6)
`TL;DR` (что делать сейчас) · **Принципы `[P]`** (вечные) · **Тактики `[T]`** (текущие, датированные) · Числа / бенчмарки · блок **«Как было раньше и что изменилось»** · Anti-patterns · Применение в анализе/оптимизации · Цитаты.

### Главные свойства базы
- **Провенанс на каждый факт.** Любое утверждение несёт ссылку `⟦video_id @дата⟧` на публичный первоисточник — можно проверить (`sources/INDEX.md`).
- **Снят конфликт «старое / новое».** Каждая тактика помечена эрой (`pre-` / `post-andromeda`). При противоречии **свежее перетирает устаревшее** (старое уходит в блок «как было» с пометкой `superseded`), а вечные принципы консолидируются по всем годам.
- **Перекрёстные ссылки.** Файлы связаны между собой (`[[категория/тема]]`) — это сеть знаний, а не разрозненные заметки.
- **Без галлюцинаций.** Числа — только из источников; заявления самой Meta помечены как `Meta-claim`.

## 🛠️ Playbooks — анализ → отчёт (6)
`audit-account` (полный аудит кабинета: что работает / что нет / почему просадка / план масштабирования) · `analyze-creative` · `analyze-video-ad` · `analyze-landing` · `analyze-settings` · `report-template` (общий формат отчёта, markdown + HTML). Каждая рекомендация привязана к конкретному `object_id`, каждая цифра — к источнику.

## 🛡️ Guardrails (2)
`ad-account-safety` — read-before-write, разделение KNOWLEDGE≠DATA, подтверждение изменений, пороги значимости, ad-eater detection. `metric-definitions` — точные определения метрик + Health Score.

## Установка
```bash
git clone https://github.com/myroslavnahirniak/meta-ads-pro ~/.claude/skills/meta-ads-pro
```
Нужен **Claude Code**. Скилл срабатывает автоматически на вопросы про Meta / Facebook / Instagram рекламу. Для работы с живыми данными нужен доступ к [Meta Marketing API](https://developers.facebook.com/documentation/ads-commerce/marketing-api). Без подключённых данных работает как консультант по методологии.

## Использование (пиши на естественном языке)
- «Проанализируй кабинет `act_XXXXXXXXX`, цель CPL = $…, бюджет $…/день — что работает, что нет, как масштабировать.»
- «Почему за неделю вырос CPL? Разложи по адсетам.»
- «Аудит настроек кампании `[id]` — что поправить под post-Andromeda.»
- «Аудит этого креатива / видео-объявления.»
- «Посмотри лендинг `[url]` — где теряем конверсию.»

## Безопасность
Аналитика — **read-only**. Любое изменение в кабинете — только после твоего подтверждения (показывает `old→new` + обоснование данными + риск). Мало данных → «недостаточно данных», без фантазий.
