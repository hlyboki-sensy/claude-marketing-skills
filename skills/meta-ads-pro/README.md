# meta-ads-pro

**AI-таргетолог по Meta (Facebook / Instagram)** на актуальній (post-Andromeda) методології **Ben Heath** — у вигляді **скіла для Claude Code**. Аналізує рекламний кабінет, креативи, відео, лендинги й налаштування → видає **дієвий звіт** і безпечно оптимізує.

**Це найповніший і найактуальніший розбір методології таргетингу в Meta, упакований у робочий AI-інструмент** — який не просто радить, а аналізує дані й діє. І не вигадує цифр.

### Формат кожного файлу (єдиний, §6)
`TL;DR` (що робити зараз) · **Принципи `[P]`** (вічні) · **Тактики `[T]`** (поточні, датовані) · Числа / бенчмарки · блок **«Як було раніше і що змінилося»** · Anti-patterns · Застосування в аналізі/оптимізації · Цитати.

### Головні властивості бази
- **Провенанс на кожен факт.** Будь-яке твердження несе посилання `⟦video_id @дата⟧` на публічне першоджерело — можна перевірити (`sources/INDEX.md`).
- **Знято конфлікт «старе / нове».** Кожна тактика позначена ерою (`pre-` / `post-andromeda`). При суперечці **свіже перетирає застаріле** (старе йде в блок «як було» з позначкою `superseded`), а вічні принципи консолідуються за всі роки.
- **Перехресні посилання.** Файли пов'язані між собою (`[[категорія/тема]]`) — це мережа знань, а не розрізнені нотатки.
- **Без галюцинацій.** Числа — тільки з джерел; заяви самої Meta позначені як `Meta-claim`.

## 🛠️ Playbooks — аналіз → звіт (6)
`audit-account` (повний аудит кабінету: що працює / що ні / чому просадка / план масштабування) · `analyze-creative` · `analyze-video-ad` · `analyze-landing` · `analyze-settings` · `report-template` (загальний формат звіту, markdown + HTML). Кожна рекомендація прив'язана до конкретного `object_id`, кожна цифра — до джерела.

## 🛡️ Guardrails (2)
`ad-account-safety` — read-before-write, розділення KNOWLEDGE≠DATA, підтвердження змін, пороги значущості, ad-eater detection. `metric-definitions` — точні визначення метрик + Health Score.
