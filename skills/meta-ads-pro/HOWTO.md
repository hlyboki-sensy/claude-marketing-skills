# meta-ads-pro — як підключити й користуватись

**Що це.** AI-таргетолог по **Meta (Facebook / Instagram)** на актуальній (**post-Andromeda**) методології Ben Heath: аналізує рекламний кабінет / креативи / відео / лендинги / налаштування → **дієвий звіт** і безпечно оптимізує. Кожен висновок — з джерела або з даних; **не вигадує цифр**.

---

## 1. Встановити (одноразово)
Скопіюй папку `meta-ads-pro/` у скіли Claude Code:
```bash
cp -R meta-ads-pro ~/.claude/skills/meta-ads-pro
```
Потрібен **Claude Code** (CLI або VS Code extension). Скіл підхопиться автоматично — тригериться, щойно мова про Meta / Facebook / Instagram рекламу. Перевірити: спитай «що вміє meta-ads-pro».

## 2. Підключити кабінет (щоб бачив живі дані)
Для роботи з реальним кабінетом потрібен доступ до [Meta Marketing API](https://developers.facebook.com/documentation/ads-commerce/marketing-api). Налаштуй доступ до **свого** рекламного кабінету (`act_…`).
> Без підключеного доступу скіл усе одно працює — як консультант по методології (радить, але без живих цифр акаунту).

- «Проаналізуй кабінет `act_XXXXXXXXX`, ціль CPL/$reg = **$…**, бюджет **$…/день** — що працює, що ні, **як масштабувати до $…/день**.»
- «Чому за тиждень виріс CPL? Розклади по адсетах і дай план.»

Скіл сам обере потрібний playbook (аудит кабінету / креативу / відео / лендингу / налаштувань) і видасть звіт із конкретними діями (паузити `ad_id=…`, дублювати концепт `ad_id=…` тощо).

## 6. Головне про методологію (щоб не здивувало)
Це **post-Andromeda** підхід — частина «класичних» порад тут свідомо змінена:
- **1 кампанія + 1 адсет**, cold+warm разом (не дробити по інтересах).
- **20+ різнопланових креативів** в адсеті; один angle = одне оголошення.
- Масштаб — наявну кампанію, **спадним %**, кроками ≥5 днів (не дублювати адсети).
- Ретаргетинг — **hybrid** (Meta й так ллє ~40% на warm); окрема warm-кампанія зазвичай шкодить.
- Тестувати **великі важелі** (offer → angle → hook), не колір кнопки; схожі креативи — через Creative Testing tool.
